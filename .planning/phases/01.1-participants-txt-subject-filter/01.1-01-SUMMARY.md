## Decisions Made

- `participants_filter.py` is a standalone utility module (not a method on `DataSourceConfig`) — keeps filter logic decoupled from config class per CONTEXT.md deferred decision
- Filter applied in `parcellate_hcp_subjects()` BEFORE the empty-list check so existing "No HCP subjects need parcellation" message still fires correctly when filter produces an empty result
- Single-subject path in `hcp_parcellation.py` intentionally excluded from filter — the flag makes no sense for an explicit single subject ID
- Hard error on missing subjects collects ALL missing subjects before raising `ValueError` to give user the complete list in one error message
